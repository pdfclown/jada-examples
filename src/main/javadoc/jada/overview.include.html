This is part of a demonstration of Jada doclet and its extension model:

  • Jada doclet (jada-core module):
    • Jada doclet configuration (see "jada-core" dependency and related configuration in root pom.xml of the jada-demo project)
    • core Javadoc tags for pre- and post-processing:
      • @jada.doc and @jada.reuseDoc — Javadoc fragments reuse.
        Useful to share content fragments, for example, horizontally among method overloads (analogous to what the standard @inheritDoc tag does vertically for inheritable elements).
      • @jada.include – external Javadoc fragments inclusion.
        Useful to share external content fragments among arbitrary Javadoc pages.
      • @jada.render – specialized Javadoc renderers
        Useful to transform data through specialized content filters.
  • JadaBiblio extension (jada-biblio module) for automatic bibliography rendering and citation linking:
    • configuration (see "jada-biblio" dependency and related configuration in root pom.xml of the jada-demo project)
    • bibliographic Javadoc tags:
      • @biblio.spec — specification reference
      • @biblio.doc – other publication reference
      • @biblio.ref – online resource reference (such as issues on an issue-tracking system)
  • JadaUML extension (jada-uml module) for UML diagrams rendering:
    • configuration (see "jada-uml" dependency and related configuration in root pom.xml of the jada-demo project)
    • UML diagrams:
      • dependency diagrams
      • package diagrams
      • class diagrams
